# Керівництво користувача PINGUI

## Запуск

```bash
./pingui.sh
```

Перший раз:

```bash
./pingui.sh --deploy
./pingui.sh
```

Відкривається вікно: **«PINGUI — Linux session route monitor»**.

## Інтерфейс

```
┌──────────────────┬────────────────────────────────────┐
│ Список цілей     │  Граф маршруту (зверху → вниз)     │
│ [ ] 8.8.8.8      │  [Your PC] → hop1 → hop2 → ціль  │
│ [✓] google.com   │  ліворуч — попередній (сірий)    │
│                  │  праворуч — поточний               │
├──────────────────┤                                    │
│ [IP або hostname]│                                    │
│ Додати Змінити   │                                    │
│ Видалити Зберегти│                                    │
├──────────────────┤                                    │
│ Статус / Лог     │                                    │
└──────────────────┴────────────────────────────────────┘
```

## Список цілей

### Чекбокс

- **Увімкнено** — worker трасує маршрут до цілі у фоні.
- **Вимкнено** — ціль лише в списку, без ICMP.
- Одночасно можна трасувати до **10** цілей (ліміт списку = ліміт активних).

### Додати

1. Введіть IPv4 або hostname у поле нижче.
2. Натисніть **Додати** або Enter.
3. Нова ціль з’являється в списку (чекбокс вимкнено).

### Змінити

1. Виберіть ціль у списку.
2. Відредагуйте текст у полі вводу (або F2 / подвійний клік по рядку).
3. **Змінити**.

### Видалити

1. Виберіть ціль.
2. **Видалити**.

### Зберегти

Записує поточний список у YAML-файл конфігу (шлях від старту, зазвичай `config/hosts.example.yaml`).
Підтвердження — рядок у логу.

## Граф маршруту

- Показується для **вибраної** цілі в списку.
- **Your PC** — локальний вузол на початку ланцюжка.
- **Hop N** — проміжний маршрутизатор; підпис з IP і середнім RTT.
- **`*`** — timeout на hop (немає відповіді).
- Кольори RTT: green (<50 ms), yellow (<150 ms), red (≥150 ms), gray — немає даних.

### Попередній vs поточний маршрут

При зміні ланцюжка IP:

- **Ліва колонка (сіра)** — попередній маршрут; для timeout показує **last known IPs**.
- **Права колонка** — поточний trace.

## Лог

- **ROUTE CHANGE** — попередження з «було / стало».
- **Error [host]** — немає прав ICMP, DNS, timeout тощо.
- Операції зі списком (додано, змінено, видалено, збережено).

## Статус-бар

«Last update [host]: HH:MM:SS» — час останнього успішного trace для вибраної цілі.

## Дані сесії

Усі маршрути та ping history — **лише RAM**. Після закриття вікна дані втрачаються.
Збережений YAML містить **лише список цілей**, не історію маршрутів.

## Типові проблеми

| Проблема | Дія |
|----------|-----|
| Не можу додати ціль | Перевірте формат IP/hostname; ліміт 10 цілей |
| Trace не працює | Увімкніть чекбокс; перевірте `./scripts/check_caps.sh` |
| Порожній граф | Увімкніть чекбокс і зачекайте перший цикл (~1 с) |
| Поле вводу сіре | Досягнуто 10 цілей — видаліть одну |

## CLI (розширені опції)

```bash
.venv/bin/python -m pingui --interval 2 --max-hops 30 --verbose
```
