ЗАдача: розробити програму для трасування маршруту і побудови карти пінгування кожної ноди маршруту зі збором статистики про зміни маршруту, пінгів тощо.

Головне рішення: Для реалізації такої системи оптимально поєднати низькорівневе відправлення ICMP-пакетів (аналог traceroute/ping) із циклічним збереженням метрик та геоданих у базу даних.
Нижче наведено структурований псевдокод у стилі Python, який розбито на логічні модулі для збору статистики, моніторингу змін та побудови карти.
------------------------------
## 1. Модуль збору метрик (Пінгування та Трасування)
Цей блок відповідає за отримання списку IP-адрес на шляху до цілі (TTL від 1 до N) та вимірювання затримки (RTT) для кожного вузла.

import time# Псевдобібліотеки для роботи з мережею та локацієюimport socket_raw_icmpimport geo_ip_provider
def trace_and_ping(target_host, max_hops=30, timeout=2.0):
    """
    Сканує маршрут до цілі та збирає поточний пінг для кожної ноди.
    Повертає список словників з інформацією про кожну ноду.
    """
    route_snapshot = []
    target_ip = socket_raw_icmp.get_host_by_name(target_host)
    
    for ttl in range(1, max_hops + 1):
        # 1. Відправка ICMP Echo Request з обмеженим TTL
        start_time = time.time()
        reply = socket_raw_icmp.send_packet(target_ip, ttl=ttl, timeout=timeout)
        rtt = (time.time() - start_time) * 1000  # Переводимо в мс
        
        if reply is None:
            # Нода не відповіла (таймаут)
            route_snapshot.append({
                "hop": ttl,
                "ip": "Timeout",
                "ping": None,
                "geo": None
            })
            continue
            
        node_ip = reply.source_ip
        
        # 2. Отримання геокоординат для карти (виконується раз на сесію або кешується)
        geo_data = geo_ip_provider.get_location(node_ip)  # {lat, lon, city}
        
        route_snapshot.append({
            "hop": ttl,
            "ip": node_ip,
            "ping": rtt,
            "geo": geo_data
        })
        
        # Якщо досягли кінцевої цілі — зупиняємо трасування
        if node_ip == target_ip:
            break
            
    return route_snapshot

## 2. Модуль аналізу змін та збереження статистики
Цей блок працює у нескінченному циклі. Він порівнює поточний маршрут із попереднім, щоб зафіксувати аномалії, зміну шляху (флапінг) або втрату пакетів.

import database_driverimport telemetry_logger
def monitor_route_lifecycle(target_host, interval_seconds=10):
    """
    Циклічно моніторить маршрут, фіксує зміни та пише метрики в БД.
    """
    db = database_driver.connect()
    previous_route_ips = []
    
    while True:
        current_time = time.time()
        # Збір поточного стану маршруту
        current_snapshot = trace_and_ping(target_host)
        current_ips = [node["ip"] for node in current_snapshot]
        
        # 1. Перевірка на зміну маршруту (Routing Change)
        if previous_route_ips and current_ips != previous_route_ips:
            telemetry_logger.warning(
                f"ALERT: Маршрут до {target_host} змінився!"
                f" Старий: {previous_route_ips} -> Новий: {current_ips}"
            )
            db.save_route_change_event(
                timestamp=current_time, 
                old_path=previous_route_ips, 
                new_path=current_ips
            )
            
        # 2. Збереження поточної статистики по кожній ноді (для графіків та карти)
        for node in current_snapshot:
            db.insert_metric(
                timestamp=current_time,
                target=target_host,
                hop=node["hop"],
                ip=node["ip"],
                ping=node["ping"],
                lat=node["geo"]["lat"] if node["geo"] else None,
                lon=node["geo"]["lon"] if node["geo"] else None
            )
            
        # Оновлюємо стан для наступної ітерації
        previous_route_ips = current_ips
        time.sleep(interval_seconds)

## 3. Модуль візуалізації (Побудова карти та графів)
Цей блок агрегує зібрані дані з бази даних та генерує візуальну карту маршруту.

import map_visualization_library  # Наприклад, Folium або Plotly
def generate_ping_map(target_host):
    """
    Генерує інтерактивну карту з маршрутом та колірним кодуванням пінгу.
    """
    db = database_driver.connect()
    # Отримуємо останній актуальний маршрут та середній пінг для кожної ноди
    nodes_data = db.get_latest_route_with_avg_pings(target_host)
    
    # Ініціалізація карти
    route_map = map_visualization_library.Map(location=[50.45, 30.52], zoom_start=4)
    path_coordinates = []
    
    for node in nodes_data:
        if not node["lat"] or not node["lon"]:
            continue  # Пропускаємо ноди без геоданих
            
        pos = [node["lat"], node["lon"]]
        path_coordinates.append(pos)
        
        # Колір маркеру залежить від стабільності/величини пінгу
        marker_color = "green"
        if node["avg_ping"] > 100: marker_color = "orange"
        if node["avg_ping"] > 250 or node["ip"] == "Timeout": marker_color = "red"
        
        # Додавання точки на карту
        route_map.add_marker(
            location=pos,
            popup=f"Hop: {node['hop']} | IP: {node['ip']} | Avg Ping: {node['avg_ping']}ms",
            color=marker_color
        )
        
    # Малювання лінії маршруту між нодами
    route_map.draw_polyline(coordinates=path_coordinates, color="blue", weight=2.5)
    
    return route_map.save_to_html("route_map.html")

## Ключові технології для реалізації проекту:

* Мережевий рівень: Бібліотека scapy (найкращий вибір для Python для створення кастомних ICMP/UDP пакетів з TTL) або стандартний модуль socket (потребує root-прав).
* База даних: TimescaleDB або InfluxDB — ідеально підходять для часових рядів (метрики пінгу), а для збереження логів змін маршруту підійде звичайний PostgreSQL.
* Візуалізація: Бібліотека Folium (генерація красивих HTML-карт на базі OpenStreetMap) або Plotly (для побудови 3D-карт та графіків коливання пінгу/jitter).
